Updated frontend for the Purchase Order System

Backend base URL used by frontend:
http://localhost:8080

Frontend run steps:
1. Open terminal in frontend folder
2. Run: npm install
3. Run: npm start
4. Open: http://localhost:3000

What was fixed:
- Rebuilt the React frontend with correct routing
- Connected pages to backend APIs for vendors, products, customers, orders, payments, and deliveries
- Fixed broken App export/import structure
- Fixed wrong API endpoint usage in frontend services
- Added dashboard summary page
- Added form validation-ready payload handling for backend DTOs
- Added clear error/success messages
- Built successfully in this environment

Note:
- Payment page uses order data for listing because the backend payment list endpoint is unstable.
- Delivery actions work only after an order has already gone through the backend approval flow and has a delivery assigned.
